# Power Trim And Extend Test Matrix

Use this matrix while implementing Stage 1. Do not mark a cell complete until the behavior is covered by focused automated tests and, for canvas-facing behavior, verified in the running app.

## Legend

- `TODO` - not implemented or not yet verified.
- `CORE` - covered by an automated core/service test only.
- `APP` - verified in the running canvas.
- `CORE+APP` - both automated and running-canvas verification are complete.
- `OPEN` - known broken or regressed.
- `N/A` - deliberately not geometrically applicable; document the safe behavior before leaving a cell as `N/A`.

Rows are the selected target entity. Columns are the cutter or extension boundary entity.

- Line target extension now has `CORE+APP` coverage for line, polyline, polygon, circle, arc, ellipse, spline, and point boundaries. The polyline boundary cell uses the built-in sample document's `bend-path`: after adding a short target line, the probe at `15,18` changed from no hit to the extended line entity. - Open polyline target extension now has `CORE+APP` coverage for line, polyline, polygon, circle, arc, ellipse, spline, and point boundaries, including both start and end extension direction handling. These entities remain valid trim targets and valid extension boundaries for open targets.
- Point entities have no span or endpoint extension direction, so their extend-target row is deliberate `N/A`. Clicking a point target remains a trim/delete operation, and point entities remain valid cutters or endpoint boundaries for applicable open targets.
- Open entities such as lines, open polylines, arcs, elliptical arcs, and splines should be extendable where a boundary intersection can be resolved.
